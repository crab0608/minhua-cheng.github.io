用 Hexo 搭建个人博客

发表于 2022-11-22 更新于 2022-12-09 分类于 工具论 阅读次数： 本文字数： 9.2k 阅读时长 ≈ 8 分钟

做个人主页其实不是件困难的事，毕竟只要把很多东西当成黑箱来用即可。所以本文分为两大部分，第一大部分 太长不看速通版 是给性急的朋友，只要按步骤操作，不求甚解的话应该不到十五分钟就可以打好基本的框架，也就是够到能用的标准，后续只需写 .md 文件即可。

但就像大多数人使用 LaTeX 一样，大多数人确实不需要知道原理，能用即可，即所谓『内容与样式相分离』原则，用户只需专注于内容，而把样式完全托付给文档类/模板；但少部分所谓『强迫症患者』或者有兴趣的人，也确实希望知其然的同时也知其所以然，后半部分即是为了满足这部分需求。

本文用 Hexo 搭建个人博客的逻辑主要如下：

  • 利用 GitHub 创建一个免费的站点 usrname.github.io
  • 利用 Hexo 在本地初始化，完成本地部署
  • 利用 Hexo 的 deploy 插件，把本地文件部署到 GitHub 上，建立本地文件和站点的联系
  • 后续只需用 hexo 创建 md 文件，并用任意编辑器写作，然后用 hexo『编译』，最后部署

关于创建站点#

进入 GitHub 账户

  1. 创建一个仓库（repository，简写为 repo），选为公开（public），名字格式为 xxx.github.io，其中 xxx 就是 GitHub 账户的用户名，例如本站就是 mathirst.github.io

  2. 点击左侧导航栏里的 Pages，回头把 branch 设为 gh-pages （现在还没创建，所以还没得选）

创建 GitHub Pages 站点#

首先在 GitHub 下要创建一个站点（website），后面命令行做的事不过是调用别人写好的模板（hexo）来配置我们的站点显示样式。这和 LaTeX 的内容与样式相分离的思想是一模一样的。

  1. 创建一个仓库（repository，简写为 repo），名字格式为 xxx.github.io，xxx 就是你想要的站点名字，比如我这里选的是 mathirst，而且如下图所示，GitHub 会自动检测是否重名（我这里是为了示范又重新打开了新建仓库的界面，由于我的站点已经存在，所以显示重名）。xxx 直接填你的 GitHub usrname。

  1. 点击左侧导航栏里的 Pages，也就是对应了 github.io 的域名，使用这个叫做 GitHub Pages 的域名不需要支付任何费用，唯二的缺点是站点的名字必须和用户名一致，而且仓库必须公开，所以要免费改站点名字就必须改用户名，不然会看到下述界面，点击 Upgrade 就会弹出付费界面，年费48刀。
